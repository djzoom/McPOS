# GUI 黄金路径检查清单

## 前置条件
- [x] 后端运行在 http://127.0.0.1:8010
- [x] .env.local 已配置
- [ ] 前端开发服务器已启动

## 测试流程

### 1. 频道总览 (Overview) → Scan
- [ ] 打开 /t2r 页面
- [ ] 点击"扫描排播"按钮
- [ ] 验证显示扫描结果（锁定集数、冲突数、资产使用情况）
- [ ] 验证 WebSocket 收到 `t2r_scan_progress` 事件

### 2. SRT 医生 → SRT检查
- [ ] 切换到"SRT 医生"标签
- [ ] 选择剧集或输入文件路径
- [ ] 点击"检查"按钮
- [ ] 验证显示重叠、间隙、编码问题
- [ ] 测试路径遍历防护（输入 `../../etc/passwd` 应被拒绝）

### 3. 描述检查 → Desc Lint
- [ ] 切换到"描述检查"标签
- [ ] 输入或选择剧集描述
- [ ] 点击"检查"按钮
- [ ] 验证显示 lint 问题和建议修复

### 4. 计划与执行 → Plan → Run
- [ ] 切换到"计划与执行"标签
- [ ] 选择剧集ID
- [ ] 点击"生成计划"按钮
- [ ] 验证显示配方（recipe）
- [ ] 点击"执行"按钮
- [ ] 验证显示执行状态和进度
- [ ] 验证 WebSocket 收到 `t2r_run_progress` 事件

### 5. 上传验证 → Upload Verify
- [ ] 切换到"上传验证"标签
- [ ] 输入剧集ID和视频ID（假数据）
- [ ] 点击"验证"按钮
- [ ] 验证显示验证结果

### 6. 审计报告 → Audit导出
- [ ] 切换到"审计报告"标签
- [ ] 选择报告类型（daily/weekly）
- [ ] 点击"生成报告"按钮
- [ ] 验证显示审计报告
- [ ] 点击"导出"按钮（如果可用）

## 验证点
- [ ] 所有API调用返回正常（无404/500错误）
- [ ] WebSocket连接正常，版本递增
- [ ] 界面响应流畅，无卡顿
- [ ] 错误处理正常（显示错误提示，不崩溃）
