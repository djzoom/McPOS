# 🤖 ChatGPT 分析开发日志 Prompt

## 📋 使用说明

将以下 prompt 复制到 ChatGPT，让它分析项目的开发日志和历史。

---

## 🎯 Prompt 内容

```
你是一位资深的软件开发项目分析师，擅长从开发日志和文档中提取关键信息、识别开发模式、发现潜在问题和提供改进建议。

请分析以下 Kat Records Studio 项目的开发日志和文档，并提供全面的分析报告。

## 📚 项目背景

Kat Records Studio 是一个自动化 Lo-Fi 音乐视频生成系统，主要功能包括：
- 从曲库自动选曲
- 生成专辑封面
- 生成标题和描述
- 混音和视频渲染
- 排播表管理
- 工作流管理

## 📊 需要分析的日志和文档

### 1. 系统日志
- `logs/system_events.log`: 系统事件日志
- `output/logs/katrec.log`: 生成操作日志

### 2. 生产日志
- `config/schedule_master.json（新架构单一数据源）`: 正式生产记录（5期）
- `config/pppschedule_master.json（新架构单一数据源）`: 完整生产记录（11期，包含DEMO）

### 3. 开发文档
- `PROJECT_CLEANUP_SUMMARY.md`: 项目清理总结（2025-11-02）
- `docs/项目综述与未来计划.md`: 过去4天开发进展（2025-10-28 至 2025-11-01）
- `docs/开发建议-下一阶段.md`: 开发路线图
- `docs/开发日志总结.md`: 开发日志总结

## 🔍 分析任务

请从以下维度进行全面分析：

### 1. 开发趋势分析
- [ ] 识别开发重点的变化趋势（从初期到现在的演变）
- [ ] 分析开发速度（代码量、功能完成度、文档完善度）
- [ ] 评估开发节奏（是否稳定、是否有爆发期或停滞期）
- [ ] 识别关键里程碑和时间节点

### 2. 功能演进分析
- [ ] 梳理核心功能的开发历程（哪些功能先实现，哪些后实现）
- [ ] 分析功能之间的依赖关系
- [ ] 识别功能实现的质量变化（从原型到稳定的过程）
- [ ] 评估功能的完整性和成熟度

### 3. 架构演进分析
- [ ] 分析架构设计的变化（状态管理、数据流、模块划分）
- [ ] 识别重构的关键节点和原因
- [ ] 评估架构设计的合理性和可扩展性
- [ ] 发现架构中的潜在问题

### 4. 代码质量分析
- [ ] 分析代码库的规模变化
- [ ] 评估代码组织结构的合理性
- [ ] 识别代码重复和冗余
- [ ] 分析测试覆盖情况

### 5. 生产效率分析
- [ ] 分析生产记录（生成了多少期内容）
- [ ] 评估系统稳定性（成功率、错误率）
- [ ] 识别瓶颈和优化机会
- [ ] 分析自动化程度的变化

### 6. 文档质量分析
- [ ] 分析文档的完善程度
- [ ] 识别文档更新的频率和质量
- [ ] 评估文档结构是否合理
- [ ] 发现文档缺失或不一致的地方

### 7. 技术债务分析
- [ ] 识别已知的技术债务
- [ ] 评估技术债务的严重程度
- [ ] 分析技术债务的积累趋势
- [ ] 提供偿还技术债务的建议

### 8. 风险和问题识别
- [ ] 识别潜在的开发风险
- [ ] 分析已遇到的问题和解决方案
- [ ] 评估系统稳定性风险
- [ ] 识别依赖和集成风险

### 9. 最佳实践识别
- [ ] 识别项目中的优秀实践
- [ ] 分析哪些做法值得推广
- [ ] 评估开发流程的合理性
- [ ] 提供改进建议

### 10. 未来规划评估
- [ ] 评估现有开发计划的合理性
- [ ] 分析优先级划分是否合理
- [ ] 识别计划中的潜在风险
- [ ] 提供优化建议

## 📝 输出要求

请提供一份结构化的分析报告，包含：

1. **执行摘要**（500字以内）
   - 项目整体状态
   - 关键发现
   - 主要建议

2. **详细分析**（每个维度一个章节）
   - 数据支撑
   - 趋势图表（文字描述）
   - 具体发现
   - 分析结论

3. **关键发现总结**
   - Top 5 关键发现
   - 最重要的趋势
   - 最需要关注的问题

4. **改进建议**
   - 短期建议（1-2周）
   - 中期建议（1-2个月）
   - 长期建议（3-6个月）

5. **风险评估**
   - 高风险项（需要立即关注）
   - 中风险项（需要注意）
   - 低风险项（持续监控）

## 🎯 分析重点

特别关注以下方面：

1. **开发效率**: 从日志中识别开发效率的变化趋势
2. **系统稳定性**: 从生产记录中分析系统的可靠性
3. **架构演进**: 理解架构设计的变化和原因
4. **技术债务**: 识别需要偿还的技术债务
5. **开发节奏**: 分析开发是否有节奏，是否可持续

## 💡 分析技巧

- 使用数据支撑观点（如代码行数、文件数、完成期数等）
- 识别模式而非孤立事件
- 关注趋势而非单点数据
- 从多个维度交叉验证结论
- 提供可执行的建议

## 🔍 特别关注

1. **工作流管理系统的实现**: 这是最近的重要功能，分析其实现质量和影响
2. **状态管理的重构**: 理解从多数据源到单一数据源的转变
3. **文档系统的整理**: 分析文档优化的效果
4. **批量生产的稳定性**: 从生产记录中分析系统在实际使用中的表现

---

请开始分析，并提供一份详细、专业的分析报告。
```

---

## 📌 使用方法

1. **复制完整 Prompt**: 将上述 prompt 完整复制
2. **粘贴到 ChatGPT**: 在 ChatGPT 对话中粘贴
3. **提供必要文件**: 如果 ChatGPT 需要查看具体文件内容，请提供相关文件的路径或内容
4. **获取分析结果**: ChatGPT 会提供结构化的分析报告

## 💡 提示

- 如果 ChatGPT 无法访问文件，你可以将关键文件的内容直接粘贴到对话中
- 可以根据需要调整 prompt，添加或删除特定的分析维度
- 建议分步骤进行分析，先做概览，再做深入分析

---

## 🔄 补充 Prompt（如果需要更深入的分析）

如果第一次分析不够深入，可以使用以下补充 prompt：

```
基于之前的分析，请进一步深入分析以下方面：

1. **开发模式识别**: 
   - 项目的开发是迭代式还是瀑布式？
   - 是否有明显的 MVP → 优化 → 扩展的开发路径？

2. **技术选型分析**:
   - 分析技术栈的选择是否合理
   - 评估依赖库的稳定性和维护性

3. **开发团队模式**:
   - 从日志中推断开发团队的规模和协作方式
   - 分析是否有明确的开发规范

4. **用户反馈循环**:
   - 从日志中识别是否有用户反馈的痕迹
   - 分析问题发现和解决的周期

5. **可维护性评估**:
   - 代码是否易于维护和扩展
   - 文档是否足够支持后续开发

请提供更深入的分析和建议。
```
