## 🔧 改进建议

### 短期（1–2周）— 必须做（P0）

#### 1. YouTube上传MVP（P0）

**功能**：
- OAuth本地回调、Token持久与刷新
- 分块上传
- 失败重试与配额退避（指数回退）
- 上传队列与状态跟踪写回`schedule_master.json`

**交付物**：
- `scripts/local_picker/youtube_upload.py`
- `config/youtube_config.yaml`
- 文档：`docs/YOUTUBE_UPLOAD_GUIDE.md`

#### 2. 日志与异常统一（P0）

**功能**：
- 统一logger工厂输出JSON行
  - 字段：`level, ts, module, episode_id, action, latency, err_code`
- 按天滚动
- 控制台与文件双通道
- 致命错误触发"回滚 + 标注error"

**交付物**：
- 更新`src/core/logger.py`增强功能
- 统一所有模块的异常处理模式
- 配置：`config/logging.yaml`

#### 3. 配置收敛（P0）

**功能**：
- `config/config.yaml` + `.env`
- 提供`config.example.yaml`和`./install.sh`一键初始化
- 所有脚本只读统一入口

**交付物**：
- `config/config.yaml`（主配置）
- `config/config.example.yaml`（模板）
- `scripts/install.sh`（初始化脚本）
- 更新`src/configuration.py`统一读取

#### 4. 核心冒烟用例（P0）

**测试覆盖**：
- 工作流三条"黄金路径"单测（单期、批量、恢复/重跑）
- 状态机转移测试（pending→…→completed/error）

**交付物**：
- 扩展`tests/test_consistency.py`
- `tests/test_workflow_golden_paths.py`

### 中期（2–6周）— P1

1. **并行化与缓存**
   - 混音/渲染多进程
   - 封面与标题生成结果缓存
   - 幂等键（episode_id + stage）
   - 失败自动重试（上限+熔断）

2. **质量门**
   - 封面/描述/标题规范化校验器
   - 生成后打分与拒收机制

3. **可观测性**
   - 指标：产出吞吐、失败率、平均时延、渲染CPU/GPU利用、上传成功率
   - 导出CSV

### 长期（>6周）— P2

1. **主题化引擎与标签系统**
2. **移动端/管理面**
3. **社区与生态**
