# 🎯 KAT Records Studio - 交互式终端使用指南

## 📋 概述

KAT Records Studio 提供了美观的交互式终端界面，通过菜单导航完成所有工作，无需记忆复杂的命令。

---

## 🚀 快速开始

### 启动交互式终端

```bash
# 方式1：使用Makefile（推荐）
make terminal

# 方式2：直接运行
python scripts/kat_terminal.py
```

### 主菜单

启动后会显示主菜单，包含以下功能：

1. 📋 **排播表管理** - 创建、查看、生成排播表
2. 🎬 **视频生成** - 生成单期或批量生成视频
3. 🔍 **查看状态** - 查看排播表、环境、API状态
4. ⚙️  **环境配置** - 初始化环境、测试编码器、配置API
5. 📚 **帮助文档** - 查看完整帮助和文档
6. 📊 **工具** - 批量操作、监视等工具

## 📚 功能详解

### 1. 排播表管理

#### 创建排播表
- 交互式输入期数、起始日期、间隔
- 自动确认和歌库记录
- 支持跳过确认（生产环境）

#### 查看排播表
- 显示全部期数或只显示pending
- 支持查看指定ID详情
- 清晰的表格展示

#### 生成完整排播
- 自动生成标题和曲目
- 更新排播表状态
- 导出Markdown格式

#### 删除排播表
- 安全确认（不可恢复）
- 仅在需要重新生成时使用

#### 监视状态
- 单次扫描或持续监视
- 自动更新completed状态
- 实时反馈

### 2. 视频生成

#### 生成单期
- 输入期数ID（YYYYMMDD格式）
- 支持测试模式
- 完整流程（封面+混音+视频+YouTube资源）

#### 批量生成
- 指定生成期数
- 自动从排播表获取pending期数
- 支持测试模式

#### 仅生成封面
- 快速生成封面和歌单
- 跳过混音和视频（用于预览）

### 3. 查看状态

#### 排播表状态
- 总期数、已完成、pending统计
- 剩余图片数量
- 详细的期数列表

#### 环境状态
- 编码器配置状态
- 环境指纹检查
- 是否需要重新初始化

#### API状态
- API密钥配置检查
- 文件权限验证
- Git追踪状态
- 可选的实际API调用测试

### 4. 环境配置

#### 初始化环境
- 自动测试所有编码器
- 选择最佳编码器
- 保存环境指纹

#### 配置API密钥
- 交互式输入（不回显）
- 自动保存到文件
- 设置安全权限

#### 测试编码器
- 手动指定测试文件
- 基准测试
- 时长验证

### 5. 帮助文档

#### 快速参考
- 常用命令速查表
- 按类别组织
- 简洁明了

#### 完整命令辞典
- 所有可用命令
- 详细参数说明
- 使用示例

#### API使用指南
- API使用场景
- 配置方法
- 成本估算

#### API安全指南
- 安全配置方法
- 最佳实践
- 密钥管理

#### 文档索引
- 列出所有文档
- 分类展示
- 快速查找

### 6. 工具

#### 批量生成
- 快速批量生成多期
- 自动使用排播表

#### 监视排播表
- 自动检测完成的期数
- 更新状态为completed

#### 分析排播表
- 曲目使用统计
- 重复分析
- 新歌使用情况

## 💡 使用技巧

### 首次使用

1. **启动终端**：
   ```bash
   make terminal
   ```

2. **查看帮助**：
   - 选择 `5. 📚 帮助文档`
   - 选择 `1. 快速参考`

3. **配置环境**：
   - 选择 `4. ⚙️ 环境配置`
   - 选择 `1. 初始化环境`
   - 选择 `2. 配置API密钥`

### 日常工作流

1. **查看排播表**：
   - 选择 `3. 🔍 查看状态`
   - 选择 `1. 排播表状态`

2. **生成单期**：
   - 选择 `2. 🎬 视频生成`
   - 选择 `1. 生成单期`
   - 输入期数ID

3. **批量生成**：
   - 选择 `6. 📊 工具`
   - 选择 `1. 批量生成`
   - 输入生成期数

### 快捷键

- `0` - 返回上级菜单或退出
- `Enter` - 使用默认值
- `Ctrl+C` - 取消操作并退出

---

## 🔧 非交互模式

如果需要快速执行命令而不进入交互界面：

```bash
# 显示帮助
python scripts/kat_terminal.py help

# 或使用命令行入口
python scripts/kat_cli.py help --quick
```

## 📝 常见问题

### Q: rich库未安装怎么办？

**A**: 运行 `pip install rich`，或使用 `make ensure-deps` 自动安装所有依赖。

### Q: 可以跳过交互式界面直接执行命令吗？

**A**: 可以，使用 `python scripts/kat_cli.py <命令>` 或 `make <target>`。

### Q: 如何退出交互式界面？

**A**: 选择 `0. 退出` 或按 `Ctrl+C`。

### Q: 界面显示异常怎么办？

**A**: 如果rich库不可用，会自动降级到简单输出模式，功能不受影响。
