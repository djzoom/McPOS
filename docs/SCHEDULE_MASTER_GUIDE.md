# 永恒排播表系统使用指南

## 📋 概述

永恒排播表系统确保：
- ✅ **排播表一旦创建，永不变更**（永恒标准）
- ✅ **图片不重复使用**（每期分配一张，用完提示）
- ✅ **标题去重**（避免"dreams of"等重复模式，节省API）
- ✅ **歌曲去重优化**（避免临近期数重复，确保起始曲目独特）

## 🚀 快速开始

### 1. 创建永恒排播表（第一步，一次性）

```bash
# 创建100期的排播表
python scripts/local_picker/create_schedule_master.py --episodes 100
```

**重要**：
- 排播表一旦创建，建议**不要修改**（除非使用 `--force` 重新创建）
- 确保图片数量 ≥ 期数（否则会报错）
- 排播表保存在：`config/schedule_master.json`

### 2. 批量生成（使用排播表）

```bash
# 批量生成10期完整内容
make 4kvideo N=10
```

系统会自动：
1. 使用排播表中分配的图片（确保不重复）
2. 检查标题模式是否重复（重复则重新生成，最多3次）
3. 避免临近期数使用相同歌曲（检查窗口：前5期）
4. 确保起始曲目独特（不与已用起始曲目重复）
5. 图片不足时给出提示

## 🔍 功能详解

### 图片管理

**分配策略**：
- 创建排播表时，随机分配图片（但顺序固定）
- 每期使用分配的图片，使用后标记
- 如果图片不足，会在创建排播表时报错

**检查剩余图片**：
```python
from scripts.local_picker.schedule_master import ScheduleMaster

master = ScheduleMaster.load()
remaining, unused = master.check_remaining_images()
print(f"剩余图片：{remaining} 张")
```

### 标题去重机制

**模式提取**：
- 移除常见词（"the", "a", "dreams", "dream"等）
- 提取前3个关键词 + 总词数
- 例如："Dreams of Midnight" → `midnight_3`（如果过滤后只剩"midnight"）

**去重流程**：
1. 生成标题后提取模式
2. 检查是否与已使用模式重复
3. 如果重复，重新生成（最多3次）
4. 优化API提示词，避免常见模式（如"dreams of"）

**节省API**：
- 使用本地模式匹配，只在必要时调用API
- 优化提示词，引导生成独特标题

### 歌曲去重优化

**避免临近期数重复**：
- 检查窗口：前5期
- 排除最近5期使用的所有歌曲
- 如果可用曲目不足26首，放宽限制

**确保起始曲目独特**：
- 追踪所有已使用的起始曲目
- 优先选择未使用过的曲目作为起始
- 确保每张专辑的起始曲目都不同

## 🔧 配置选项

### 创建排播表参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--episodes` | 总期数（必需） | - |
| `--start-date` | 起始日期 (YYYY-MM-DD) | 2025-11-01 |
| `--interval` | 排播间隔（天） | 2 |
| `--images-dir` | 图片目录 | assets/design/images |
| `--force` | 强制覆盖已存在 | False |

### 批量生成参数

| 参数 | 说明 |
|------|------|
| `N` | 生成期数（必需） |
| `DEMO` | 测试模式（使用DEMO文件夹） |

## ⚠️ 注意事项

### 1. 排播表一旦创建，建议不要修改

- 排播表是"永恒标准"，修改会影响已生成内容的追溯性
- 如需重新创建，使用 `--force` 参数

### 2. 图片数量必须 ≥ 期数

- 创建排播表时会检查
- 如果不足，需要补充图片后重新创建

### 3. 标题去重可能增加API调用

- 如果连续3次都重复，会使用最后一次生成的标题
- 建议优化提示词或增加词汇多样性

### 4. 歌曲去重可能影响随机性

- 如果可用曲目很少，会放宽限制
- 确保曲库有足够的多样性

## 🔄 工作流程

```
1. 创建永恒排播表
   ↓
2. 批量生成（自动使用排播表）
   ↓
3. 每期生成时：
   - 使用分配的图片
   - 检查标题模式（去重）
   - 检查歌曲（避免临近期重复）
   - 确保起始曲目独特
   ↓
4. 更新排播表状态
```

## 🐛 故障排除

### 问题：找不到排播表

**解决**：
```bash
# 创建排播表
python scripts/local_picker/create_schedule_master.py --episodes 100
```

### 问题：图片不足

**解决**：
1. 检查图片目录：`assets/design/images/`
2. 补充图片后重新创建排播表（使用 `--force`）

### 问题：标题重复过多

**解决**：
1. 检查已使用的标题模式
2. 优化API提示词（在 `_try_openai_title` 中）
3. 增加词汇多样性

### 问题：歌曲可用数量不足

**解决**：
1. 检查曲库规模
2. 系统会自动放宽限制（如果可用 < 26首）
3. 考虑扩充曲库
