## 四、大规模信息节制策略

### 设计原则

> **"秒找、秒判、秒处理"** - 100频道下依然流畅

### 实现策略

#### 1. 虚拟滚动（React Virtuoso）

**适用场景**：
- 频道列表 > 20 项
- 队列任务 > 50 项
- 期数列表 > 100 项

**实现**：
```typescript

<Virtuoso
  data={channels}
  itemContent={(index, channel) => <ChannelCard channel={channel} />}
  style={{ height: '600px' }}
  // 只渲染可见区域，性能优化
/>
```

#### 2. 关注集合（Watchlists）

**功能**：用户可创建自定义频道分组，快速切换关注集合

```
[我的关注集合 ▼]
├─ 🔥 核心频道 (5)
├─ 📊 数据分析 (3)
├─ ⚠️ 问题频道 (2)
└─ [创建新集合...]
```

**特性**：
- 支持多个关注集合
- 快速切换视图
- 集合持久化保存

#### 3. 密度切换

**三种密度模式**：

| 模式 | 适用场景 | 卡片尺寸 | 显示信息 |
|------|---------|---------|---------|
| **舒适** | 10频道以下 | 大 | 完整信息+图表 |
| **标准** | 10-50频道 | 中 | 核心信息 |
| **紧凑** | 50+频道 | 小 | 关键指标 |

**切换方式**：
```
[密度: 舒适 ▼]
├─ 舒适（推荐 <10频道）
├─ 标准（推荐 10-50频道）
└─ 紧凑（推荐 50+频道）
```

#### 4. 事件抖动合并

**问题**：100频道同时更新时，频繁的WebSocket消息会导致界面闪烁

**解决方案**：
```typescript
// 防抖合并更新
const debouncedUpdate = useMemo(
  () => debounce((updates) => {
    // 批量更新状态
    batch(() => {
      updates.forEach(update => {
        applyUpdate(update)
      })
    })
  }, 300), // 300ms内合并所有更新
  []
)
```

**效果**：
- 300ms内的多个更新合并为一次
- 减少重渲染次数
- 保持界面流畅

#### 5. 信息分层显示

**默认显示**：核心指标（状态、下次发片、队列数）  
**悬停展开**：详细信息（进度、历史、操作菜单）  
**点击详情**：完整信息页面

**原则**：**"先看再点，先概览再深入"**

##### 3. 统一刷新逻辑 ⭐⭐⭐

**目标**：保证前端数据一致性，统一刷新机制

**实现方案**：
```typescript
// hooks/useUnifiedRefresh.ts
export const useUnifiedRefresh = () => {
  const queryClient = useQueryClient()
  const wsConnected = useWebSocketStore(state => state.connected)
  
  const refreshAll = async () => {
    // 1. 显示加载状态
    setRefreshing(true)
    
    // 2. 失效所有查询
    await queryClient.invalidateQueries()
    
    // 3. 如果WebSocket断开，触发一次全量拉取
    if (!wsConnected) {
      await Promise.all([
        queryClient.refetchQueries(['channels']),
        queryClient.refetchQueries(['queue']),
        queryClient.refetchQueries(['episodes']),
      ])
    }
    
    setRefreshing(false)
    toast.success('数据已更新')
  }
  
  return { refreshAll, isRefreshing: refreshing }
}
```

**全局刷新按钮**：
```typescript
<button onClick={refreshAll} disabled={isRefreshing}>
  {isRefreshing ? <Spinner /> : <RefreshIcon />}
  刷新
</button>
```

**验收标准**：
- ✅ 统一刷新按钮在所有页面可用
- ✅ 刷新状态全局显示
- ✅ WebSocket断开时自动回退到轮询
- ✅ 刷新完成后Toast提示

##### 6. Toast错误提示 ⭐⭐

**目标**：取代console报错，用户友好的错误提示

**实现方案**：
```typescript
// 使用 ShadCN Toast 或 react-hot-toast
import toast from 'react-hot-toast'

// 统一错误处理
const handleError = (error: Error, context?: string) => {
  console.error(context || 'Error:', error)
  toast.error(
    error.message || '操作失败，请稍后重试',
    {
      duration: 5000,
      position: 'top-right',
    }
  )
}

// 使用
try {
  await retryTask(taskId)
  toast.success('任务已重新排队')
} catch (error) {
  handleError(error, '重试任务失败')
}
```

**Toast类型**：
- ✅ 成功（绿色）
- ⚠️ 警告（黄色）
- ❌ 错误（红色）
- ℹ️ 信息（蓝色）

**验收标准**：
- ✅ 所有API错误显示Toast
- ✅ Toast自动消失（5秒）
- ✅ 支持手动关闭
- ✅ 多个Toast堆叠显示

## 八、设计原则总结

### 1. 信息密度可控

- **舒适模式**：10频道，完整信息展示
- **标准模式**：10-50频道，核心信息
- **紧凑模式**：50+频道，关键指标

### 2. 操作路径最短

- **一键操作**：批量重试、批量暂停
- **拖拽交互**：时间线改期、排序
- **上下文菜单**：右键快速操作

### 3. 状态即时反馈

- **实时更新**：WebSocket推送
- **加载状态**：Skeleton + 进度条
- **错误处理**：Toast + 重试按钮

### 4. 大规模可扩展

- **虚拟滚动**：支持千级列表
- **事件合并**：防抖优化
- **关注集合**：个性化视图
