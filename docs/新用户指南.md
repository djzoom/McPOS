## 🎯 项目简介

### 这个项目能做什么？

**Kat Records Studio** 是一个自动化内容生产系统，可以帮助您：

1. ✅ **自动合成音乐节目视频**
   - 从音乐库自动选择歌曲
   - 生成封面图片
   - 混合音频
   - 渲染4K视频

2. ✅ **检查和修复文件**
   - 检查字幕文件（SRT）的时间轴问题
   - 检查视频描述的SEO和规范
   - 检测重复使用的素材

3. ✅ **自动上传到YouTube**
   - 自动上传视频
   - 自动填写标题和描述
   - 验证上传结果

### 核心概念

- **排播表**：一份计划文件，列出所有要制作的节目和发布日期
- **期数（Episode）**：每个节目都有一个唯一的ID，格式如 `20251102`（年月日）
- **工作流阶段**：制作一个节目需要经过多个阶段（选曲→生成封面→混音→渲染→上传）

## 🚀 快速开始

### 前置要求

在开始之前，请确保您的电脑已安装：

1. **Python 3.11** 或更高版本
   - 检查方法：打开终端（Terminal），输入 `python3 --version`
   - 如果未安装，请访问 [python.org](https://www.python.org/) 下载

2. **Node.js 和 pnpm**（仅Web界面需要）
   - 检查方法：`node --version` 和 `pnpm --version`

3. **FFmpeg**（用于视频处理）
   - 检查方法：`ffmpeg -version`

### 首次设置

#### 步骤1：配置环境

```bash
# 1. 进入项目目录
cd /Users/z/Downloads/Kat_Rec

# 2. 创建虚拟环境（如果还没有）
python3 -m venv .venv311

# 3. 激活虚拟环境
source .venv311/bin/activate  # macOS/Linux
# 或
.venv311\Scripts\activate     # Windows

# 4. 安装依赖
pip install -r requirements.txt
```

#### 步骤2：配置YouTube上传（可选）

如果您需要自动上传到YouTube：

```bash
# 运行配置向导
python scripts/local_picker/youtube_auth.py --setup
```

按照提示在浏览器中完成授权即可。

#### 步骤3：选择使用方式

## 🖥️ 三种使用方式

### 方式1：Web界面（推荐新手）✨

**适合人群**：不熟悉命令行，喜欢图形界面

#### 启动步骤

1. **启动后端服务**（打开一个新的终端窗口）：

```bash
cd /Users/z/Downloads/Kat_Rec/kat_rec_web/backend
source ../../.venv311/bin/activate
export USE_MOCK_MODE=false
uvicorn main:app --reload --port 8010
```

等待看到类似这样的提示：
```
INFO:     Uvicorn running on http://127.0.0.1:8010
✅ T2R/MCRB routers registered
```

2. **启动前端界面**（打开另一个终端窗口）：

```bash
cd /Users/z/Downloads/Kat_Rec/kat_rec_web/frontend
pnpm install  # 仅首次需要
pnpm dev
```

等待看到：
```
✓ Ready on http://localhost:3000
```

3. **打开浏览器**，访问：`http://localhost:3000/t2r`

#### 界面功能说明

Web界面包含8个标签页：

| 标签 | 功能 | 说明 |
|------|------|------|
| 📊 **频道总览** | 扫描排播表 | 查看已锁定节目、冲突数量 |
| 📅 **排播医生** | 排播表管理 | 查看和修改排播表 |
| 🖼️ **资产健康** | 素材检查 | 查看图片和歌曲使用情况 |
| 📝 **SRT医生** | 字幕检查 | 检查并修复字幕时间轴问题 |
| ✍️ **描述检查** | 描述规范化 | 检查视频描述是否符合规范 |
| ⚙️ **计划与执行** | 生成和执行节目 | 生成节目计划，一键执行 |
| ✅ **上传验证** | 验证上传结果 | 检查已上传视频的状态 |
| 📋 **审计报告** | 生成报告 | 导出日报、周报等 |

#### 基本操作流程

1. **扫描排播表**：点击"频道总览"标签，点击"执行扫描"按钮
2. **生成节目计划**：点击"计划与执行"标签，输入期数ID（如 `20251102`），点击"生成 Recipe"
3. **执行制作**：点击"一键执行"按钮，系统会自动完成所有阶段
4. **检查结果**：在"上传验证"标签中验证上传状态

### 方式2：命令行界面（适合批量操作）💻

**适合人群**：需要批量处理，熟悉基本命令行操作

#### 启动交互式界面

```bash
python scripts/kat_terminal.py
```

这会打开一个菜单界面，您可以通过方向键和回车键选择功能。

#### 常用命令

##### 1. 创建排播表

```bash
python scripts/kat_cli.py schedule create --episodes 100
```

这会创建一个包含100期节目的排播表，默认从今天开始，每2天发布一期。

##### 2. 生成单期节目

```bash
python scripts/kat_cli.py generate --id 20251102
```

这会生成期数为 `20251102` 的完整节目（包括封面、音频、视频）。

##### 3. 批量生成

```bash
python scripts/kat_cli.py batch --count 10
```

这会自动生成排播表中前10个待制作的节目。

##### 4. 上传到YouTube

```bash
python scripts/kat_cli.py upload --episode 20251102
```

##### 5. 查看排播表

```bash
python scripts/kat_cli.py schedule show
```

##### 6. 检查节目文件

```bash
python scripts/local_picker/check_episode_files.py --episode-id 20251102
```

### 方式3：桌面应用（最便捷）🖥️

**适合人群**：希望一键启动，不需要打开终端

#### 安装桌面应用

1. **安装Tauri依赖**：

```bash
cd desktop/tauri
pnpm install
```

2. **构建前端**：

```bash
cd ../../kat_rec_web/frontend
NEXT_OUTPUT_MODE=export pnpm build
```

3. **启动应用**：

```bash
cd ../../..
make app:dev
```

应用会自动：
- 启动后端服务
- 等待后端就绪
- 打开应用窗口并导航到 `/t2r` 页面

## 📋 完整工作流程

### 场景1：首次使用 - 从零开始制作节目

#### 第一步：创建排播表

**Web界面方式**：
1. 打开"排播医生"标签
2. 点击"创建新排播表"
3. 输入期数（例如：100）
4. 确认创建

**命令行方式**：
```bash
python scripts/kat_cli.py schedule create --episodes 100
```

#### 第二步：生成节目

**Web界面方式**：
1. 打开"计划与执行"标签
2. 输入期数ID（例如：`20251102`）
3. 点击"生成 Recipe"
4. 等待生成完成
5. 点击"一键执行"

**命令行方式**：
```bash
# 生成单期
python scripts/kat_cli.py generate --id 20251102

# 或批量生成10期
python scripts/kat_cli.py batch --count 10
```

#### 第三步：检查文件

**Web界面方式**：
1. 打开"SRT医生"标签
2. 输入期数ID
3. 点击"检查"
4. 如有问题，点击"修复"

**命令行方式**：
```bash
python scripts/local_picker/check_episode_files.py --episode-id 20251102
```

#### 第四步：上传到YouTube

**Web界面方式**：
1. 打开"上传验证"标签
2. 输入期数ID和视频ID
3. 点击"验证"

**命令行方式**：
```bash
python scripts/kat_cli.py upload --episode 20251102
```

## ❓ 常见问题

### Q1: 如何知道我的节目是否成功生成？

**方法1：检查输出目录**
```bash
ls output/
```
如果看到以期数ID命名的文件夹，说明生成成功。

**方法2：查看Web界面**
打开"频道总览"，点击"执行扫描"，查看"已锁定"数量。

**方法3：检查排播表状态**
```bash
python scripts/kat_cli.py schedule show --id 20251102
```

### Q2: 如何重新生成某个节目？

如果您想重新生成某个已完成的节目：

**命令行方式**：
```bash
# 删除旧文件（可选）
rm -rf output/20251102_*

# 重新生成
python scripts/kat_cli.py generate --id 20251102 --force
```

**Web界面方式**：
1. 打开"排播医生"标签
2. 找到对应期数
3. 点击"重置状态"
4. 在"计划与执行"中重新执行

### Q3: 如何检查字幕文件是否有问题？

**Web界面方式**（推荐）：
1. 打开"SRT医生"标签
2. 输入期数ID或上传SRT文件
3. 点击"检查"
4. 如有问题，点击"修复"

**命令行方式**：
```bash
# 检查特定期数
python scripts/local_picker/check_episode_files.py --episode-id 20251102

# 检查所有期数
python scripts/local_picker/check_episode_files.py
```

### Q4: 如何查看上传状态？

**Web界面方式**：
1. 打开"上传验证"标签
2. 输入视频ID和期数ID
3. 点击"验证"

**命令行方式**：
```bash
# 查看上传日志
tail -f logs/katrec.log | grep upload

# 查看排播表中的上传状态
python scripts/kat_cli.py schedule show --id 20251102
```

### Q5: 遇到"端口已被占用"错误怎么办？

**解决方法1：更改端口**
```bash
# 后端改为8010端口（已在代码中配置）
uvicorn main:app --port 8010

# 前端改为3001端口
pnpm dev --port 3001
```

**解决方法2：关闭占用端口的进程**
```bash
# 查找占用8000端口的进程
lsof -ti:8000

# 结束进程
kill -9 $(lsof -ti:8000)
```

### Q6: 如何配置YouTube自动上传？

1. **首次配置**：
```bash
python scripts/local_picker/youtube_auth.py --setup
```

2. **按提示操作**：
   - 系统会打开浏览器
   - 登录您的Google账号
   - 授权应用访问YouTube
   - 复制授权码回终端

3. **验证配置**：
```bash
python scripts/check_youtube_api.py
```

如果看到"✅ YouTube API配置正确"，说明配置成功。

### Q7: 如何查看系统运行状态？

**Web界面方式**：
打开"频道总览"标签，查看：
- 已锁定数量
- 冲突数量
- 总期数
- 重复图片数量

**命令行方式**：
```bash
# 快速状态检查
bash scripts/quick_probe.sh

# 详细状态
curl http://127.0.0.1:8010/metrics/system | jq .
```

## 🎓 进阶功能

### 1. 自定义排播间隔

创建排播表时指定间隔：

```bash
python scripts/kat_cli.py schedule create \
  --episodes 100 \
  --start-date 2025-12-01 \
  --interval 3
```

这会在2025年12月1日开始，每3天发布一期。

### 2. 只生成特定阶段

如果只想生成封面和音频，不生成视频：

```bash
python scripts/kat_cli.py generate --id 20251102 --no-video
```

### 3. 生成审计报告

**Web界面方式**：
1. 打开"审计报告"标签
2. 选择报告类型（日报/周报/自定义）
3. 选择格式（JSON/CSV/Markdown）
4. 点击"生成报告"
5. 点击"导出"保存文件

**命令行方式**：
```bash
curl "http://127.0.0.1:8010/api/t2r/audit?report_type=daily&format=markdown" \
  -o daily_report.md
```

### 4. 批量修复所有SRT文件

**Web界面方式**：
1. 打开"SRT医生"标签
2. 系统会自动列出所有有问题的SRT文件
3. 点击"批量修复"

**命令行方式**：
```bash
# 检查所有SRT
python scripts/local_picker/check_episode_files.py

# 手动修复特定期数（需要查看输出后手动调用修复）
```

### 5. 监控系统运行

**实时监控**：
```bash
python scripts/local_picker/watch_schedule_status.py --watch
```

这会持续显示排播表状态更新。
